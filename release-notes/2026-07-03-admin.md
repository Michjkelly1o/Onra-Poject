1 Customer Management
The customer list + detail screen where the owner manages every customer and how they hear from the studio.
Capabilities:
Marketing preferences — Expanded from 3 opt-ins to 8 on the customer Details tab. Four delivery channels (Marketing emails, Marketing WhatsApp, Marketing SMS, Push notifications) and four content topics (Studio announcements, New class launch, Special offers, Promo code offers). Each row shows Subscribed with a green check or Unsubscribed with a red cross so the owner sees at a glance what a customer wants to receive.
Realistic customer counts — The demo now runs on around 1,500 customers instead of a small handful, so opt-in counts, dashboard tiles, and every "how many customers" number in the app reads like a real studio.
All locations default — The branch filter at the top of the customer list opens on "All locations" so the module lands on the aggregate view. Drill into a specific branch when needed.

2 General Settings
The Settings hub reorganised into three groups so the owner spends less time hunting for a screen.
Capabilities:
Three-group organisation — Every settings screen now sits under one of three tabs: Business (Business & locations, Branding), Operations (Booking rules, Tax, Integrations), or Customer (Customer notifications, Agreements). The tab bar sits at the top of each screen so the owner always sees where they are within the group.
Sidebar dropdown — Clicking the Settings gear at the bottom of the sidebar opens a small dropdown showing Business, Operations, and Customer with their own icons. Same pattern as the profile chip.
Tax module — Added a Tax Registration Number (TRN) card at the top of the VAT tab with the studio's TRN number, issuing country, and a toggle to show it on customer invoices. Every tax rate now carries an effective window (Valid from and Valid until) shown as a new column on the list ("DD/MM/YYYY - DD/MM/YYYY" or "Ongoing" when open-ended). The Add / Edit tax rate screen picked up matching date pickers at the bottom.
Agreements module — New Last signed column on the agreements list showing when each agreement was most recently accepted by anyone. Opening an agreement now lands on the Acceptance status view — the three big numbers (Signed on current version / On older version / Pending or never signed) plus the customer table are the first thing the owner sees.
Payroll — Instructor detail page renamed to "Payroll Details" so the title matches what the screen actually shows. Tax withholding on payroll is now country-aware: hidden for UAE and other GCC studios, reappears automatically if the studio switches to a country that does apply income tax. The payroll CSV export now breaks each instructor down into typed rows (Pay model, Component, Basis, Rate, Amount) with an explicit Subtotal for multi-component pay models, replacing the free-text explanation column that used to be hard to read in Excel.
Customer notifications — Payment notification rows are now protected: the owner can turn any single channel off, but the last remaining channel stays locked on so a paying customer always gets told. The WhatsApp column, approval status pills, and the entire message template editor (Email / WhatsApp / SMS / Manage timing tabs) update live based on whether the WhatsApp Business integration is connected. Marketing category shows a live opt-in banner ("X of Y customers have opted in") that tracks the customer preferences above.

General updates
Sidebar accordion — Clicking a menu that has a submenu now automatically collapses every other expandable group. Only one submenu stays open at a time so the sidebar reads clean regardless of where the owner is.
Sub-menu hierarchy — Sub-menu rows sit clearly indented under their parent group with a consistent visual step, so which module a screen belongs to reads at a glance.
Branch filter defaults — Every list module now defaults its branch filter to "All locations" — Customers, Promotions, Marketing, Schedule, POS, Dashboard, Pay rates, and In-app notifications all open on the aggregate view. Pick a specific branch when you need to drill in.
Module renames — "Promo codes" is now "Promotions" everywhere the module surfaces (sidebar, page title, toolbar, toasts). The role label on the account/profile screen reads "Owner" instead of "Super admin" so the persona matches what the demo represents.
