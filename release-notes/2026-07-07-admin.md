1 Dashboard
The owner's landing screen — three tabs that split the studio's numbers between what's happening now, what's coming up, and what's trending over time.
Capabilities:
New "Coming up" tab — A third tab sits between Today at glance and Performance. Shows six look-ahead cards on a single row: Upcoming recurring revenue, Bookings ahead, Renewals due, Failed payments, At-risk clients, and Under filled classes.
Next 7 / Next 30 days pill — A pill next to the location dropdown flips the window. Cards recompute instantly for either horizon; "Recoverable now" failed payments and "no visit 14-30 days" at-risk are always in view.
Click through to the fix — Renewals due, Failed payments, At-risk clients, and Under filled classes each open the drill-through panel with the exact list behind the count. The other two cards stay read-only.
Failed payment row goes to the customer — Any row in the Failed payments panel now opens that customer's Payment history tab directly, so the fix is one click away.
At-risk row goes to the customer — Same pattern in the At-risk clients panel: click a row, land on the customer's profile.
"Needs attention today" removed — The old list at the bottom of Today at glance is gone. The four attention items live on the Coming-up tab cards instead, so there's one place to look, not two.
Counts always match the list — If a card says "5 failed payments" the drill-through panel shows exactly those 5, at both pill settings. Same for Under-filled classes and Renewals due.

2 Customer Management
The customer list + detail screen where the owner tracks each customer's plan, credits, and activity.
Capabilities:
Credit left column — The Plan tab now has a "Credit left" column between Plan type and Status. A green progress bar fills to the amount left with the count on the right (e.g. "9/10"). Unlimited memberships show "Unlimited" with a full bar.
Muted when finished — On Expired and Removed rows, only the Credit-left cell dims (gray text + gray bar) so the row still reads normally elsewhere. Frozen and Cancelled stay full-strength since they can still be acted on.
Sidepanel matches the table — The "Total credits" summary card in the sidepanel now always equals the sum of the Credit-left column. If the table shows 12/12 the sidepanel reads 12 credits left, never 0.
Real credit caps everywhere — Fixed an issue where some customers with an active membership showed "0/0" instead of "0/10". The real per-plan cap (10, 12, 20, or Unlimited) now flows through every screen that shows credits.

3 Customer Notifications
The settings screen where the owner controls every message the customer receives.
Capabilities:
Critical on any event — A new "Condition" tab on the Edit template modal lets the owner flip any event to Critical, not just Payments. A Critical pill shows on the row header once it's on, matching what Payment events already do.
Auto-enables Email when needed — Turning Critical ON when no channel is enabled automatically switches Email on, so the "at least one channel stays live" rule is honoured from the moment the flip happens. A toast confirms what was auto-enabled.
Last-channel lock, everywhere — The rule that used to only guard Payment rows now guards every Critical row: the owner can turn any channel off, but the last remaining channel is locked. Trying to switch it off fires an amber toast explaining why.

4 Business & Locations
The settings screen where the owner manages branches, rooms, and working hours.
Capabilities:
Enable moved into row actions — The dedicated Enable column on the Locations & Rooms table is gone. Enable and Deactivate now live in the row's kebab menu, matching how Memberships & Packages and Promotions work. Confirmation modals + the underlying logic are unchanged.
Working days polish — Working days in the branch card render consistently. Days show red only when the owner marks them closed; days left open stay in the default colour.

5 Insights
The single analytics landing — one screen for every KPI the studio tracks.
Capabilities:
KPI page becomes Insights — What was called "KPI" is now "Insights" in the sidebar. The old Insights page is retired; the Insights label goes to the new, richer surface.
Four category tabs — Financial, Client, Class, and Marketing. Each tab carries its own set of KPI cards computed live from the studio's real numbers (transactions, bookings, plans, referrals, campaigns).
Same numbers as Reports — Insights cards and Financial reports read from the exact same data layer. A number on an Insights card matches the corresponding Report row every time.
Date range picker + search — Pick "This week", "This month", "Last 30 days" or a custom range from a dropdown. Every card recomputes for the range. Search filters the visible KPIs by name.
Live updates — A POS sale, a booking, a plan change, a refund — the Insights cards on screen recompute in the same moment. No refresh.

General updates
Image banner size unified — Create and Edit screens on Class templates, Class schedule, Services, Promotions, and Campaigns all now show the image banner as a 4:3 tile with a compact right-side label and Upload button. Same layout everywhere.
Card lists 4 per row — Class templates, Promotions, and Campaigns show four cards per row instead of three, so the grid fits more without extra scrolling.
Compact date on Promotion + Campaign cards — "Valid until" now reads "31/12/2026, 12:00 AM" instead of the long month name, so the cards don't wrap on the four-column grid.
Categories moved to Services & pricing — The Categories submenu now sits under Services & pricing next to Memberships & packages and Services. Same page, cleaner grouping.
Booking Rules waitlist copy tidy — The waitlist explanation reads "After cut off, a freed spot:" (dropped the trailing "to").
Reports scope for this release — The Reports landing shows the Financial category only for this demo push. The other five categories are being finished and will land in an upcoming release.
