1 Booking Rules
The settings screen where the owner sets how customers can book classes, join a waitlist, and cancel — all in one place.
Capabilities:
Booking window — Set how far in advance customers can book and whether to allow last-minute bookings right up to class start. When last-minute booking is on, the cutoff field hides itself so the setting stays clean.
Waitlist — Master toggle at the top of the card. When on, the owner sets the maximum waiting spots, which channels to notify (WhatsApp, Email, SMS, Push), what happens when a spot opens (auto add the next person or send a notification), and what happens after the auto-promotion cutoff (reopen for anyone, keep the waitlist order, or leave the spot empty). When off, the whole card collapses to a single row.
Cancellation policy — Two-tab card with underline tabs: "Cancellation rule" shows credit and package member rules plus membership fee toggles at a glance; "Applied to" lists which packages and classes the policy covers.
Live-linked windows — The "Match the free cancellation window" toggle in the Waitlist panel automatically reads the same hours as the cancellation policy. Change the cancellation window and the waitlist auto-promotion cutoff follows in the same render.
Applies to any packages or classes — The cancellation policy only fires on bookings for the products the owner selected. Others use the studio default. The list is searchable with a filter chip.

2 Agreements
The settings screen where the owner manages liability waivers, terms, and any legal document members must sign before booking.
Capabilities:
Redesigned list — New columns show branch coverage (which branches each agreement applies to) and adoption coverage (percentage of active members who have signed, with an amber warning when re-acceptance is needed).
Effective dates — On create or edit, pick "Ongoing" (no expiry) or "Set an expiry date" (with issue + expiry pickers). Ongoing agreements display an "Ongoing" pill in the list and detail page; dated ones show the expiry date.
Re-acceptance rule — When a new version publishes, existing signed members automatically flip to "Re-accept due" and are prompted to accept the new version before their next booking.
Minors and guardian consent — Toggle a rule requiring a guardian signature for members under 18. Detail page shows the current state with a tooltip explanation.
Acceptance status tab — Three summary cards at the top (Signed current version, On older version, Pending or never signed) followed by three sub-tabs (All signed, Needs re-acceptance, Pending or never). Each row has a "View" action that deep-links to the customer detail page.
Customer detail Agreements tab — Status is now split into three distinct pills: Signed (green), Re-accept due (amber), Never signed (red). "Unsigned" has been renamed to "Never signed" for clarity.

3 Payroll
The screen where the owner processes payroll for instructors and exports the payout data for their accountant.
Capabilities:
Process Payroll modal — The summary now reads "Gross wages · Tax withholding · Net payout" with honest math (the three lines actually add up). Renamed from the old "Gross revenue / Tax rate / Total" that mixed studio and payroll concepts.
Live tax withholding — The withholding percentage is read live from Settings → Tax → Pay rate rule. Change the rate in the Tax module and the payroll modal updates in the same render. No more hardcoded zero.
Export enrichment — The CSV export gains three new columns so anyone can audit the calculation:
Class rate (AED) — the AED-per-class figure that drove each row.
Percentage (%) — filled only for Split Rate and revenue-split Hybrid rates. Empty for Flat, Tiered, Monthly, and attendance-bonus Hybrid rates.
Notes / Explanation — a one-line plain-language breakdown with real numbers, for example "30% of AED 6,750 revenue + AED 15 × 45 customers = AED 2,700".
Class revenue base column — The old "Gross revenue" column is now labelled "Class revenue base" across the payroll table, the metric card, and the CSV. It reads as the reference figure that Split Rate calculations use, not the instructor's take-home. This matches the fix already made on the confirmation modal.
Consistent naming across the screen — Table sortable header, top metric card, admin compensation landing card, and CSV all say "Class revenue base" together so nothing looks out of step.

General updates
Class revenue base everywhere — The "Gross revenue" wording that used to appear on payroll surfaces is renamed to "Class revenue base" on every payroll screen (Run payroll table, top metric cards, admin compensation landing, and the CSV export). Consistent naming, no more mismatch.
Tax module now feeds Payroll — When the owner configures a Pay rate tax rule in Settings → Tax, the Process Payroll modal picks it up automatically. One place to edit, all screens update together.
Shared side-panel patterns — The Booking Rules side panels reuse the same visual language and side-panel behaviour as the Referral module panels (slide-in from the right, edit-and-save, cancel to discard). Owners moving between modules find the same familiar chrome each time.
Reusable list accordion — The "Applies to" list in the Booking Rules cancellation policy panel uses the same accordion pattern the Agreements form uses for applicable services. One pattern, consistent behaviour everywhere it appears.
