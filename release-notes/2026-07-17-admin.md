Note:
This push tightens the Dashboard's Performance tab, folds Freeze policy into Booking rules, adds a password row on the customer profile that mirrors what the member uses on the customer app, and locks the referral program to a single reward type end to end. Today's feedback from the meeting is not tackled yet in this push — we only ship what was completed this week. In the prototype you may still see some 404 pages; that means the page/module/feature is still being developed.
New side for admin: to navigate to admin please change the link to "https://onra-poject.vercel.app/admin/dashboard" and for the instructor side, please change it back to "https://onra-poject.vercel.app/instructor/dashboard"
Interactive experience — This is a fully interactive prototype. Feel free to click, create, edit, and delete items to test the workflows.
Live updates — The prototype uses realistic sample data. Any changes you make apply live as you navigate between different screens.
Session persistence — Any data you create, edit, or cancel survives a page refresh and tab close. Your demo session sticks until you explicitly reset.

1 Dashboard
The three-tab dashboard — Today, Coming Up, and Performance.
Capabilities:
Rounded pills — The session-type filter (All / Class / Private / Recovery) on Today and Coming Up now uses rounded pill shapes for a softer feel that matches the rest of the app.
Performance metrics renamed and period-scoped — The four metric cards on the Performance tab now read simply as Revenue, Members, Classes, and Bookings. Values reflect the selected range (this week / this month) instead of only today's number, and the small "vs" caption under each metric updates to match — "vs last week", "vs last month" — instead of a fixed label.
Metric titles cleaned up — Every metric title across the three tabs no longer bakes in a time word like "today" or "week". The range picker at the top is the single source of truth for the period, so the titles read cleanly whichever range is selected.

2 Settings
Studio-wide policies and rules.
Capabilities:
Freeze policy moved under Booking rules — The freeze policy is now the fourth section under Settings → Booking rules, sitting alongside the cancellation policy. The stand-alone Freeze policy tab is retired. The rules, per-branch override behavior, and charge-now flow are unchanged — this is a placement change only.
Referral is one reward type at a time — The referral setup now enforces a single reward type — either Class credit or Account credit, not both. Picking one auto-selects the other and resets both amounts, so a credit-count value can never accidentally save as an AED value. A short hint on the panel spells out the rule for the admin.

3 Customer
Member profiles + list.
Capabilities:
Password on the profile — The Sign in credentials section on the customer profile now shows a Password row, view-only for admin. Click the eye icon to reveal it. This is the same password the member uses to sign in on the customer app, so admin can help with sign-in questions at a glance.

4 Marketing & Promotions
Campaign and promo builders.
Capabilities:
Image size guide — Both the campaign creation form and the promotion creation form now show a semibold "Image size guide" label with a help icon next to the banner uploader. Hover the icon for the recommended dimensions and ratio, so the thumbnail lands crisp on the customer app instead of being cropped hard.

General updates
Cross-module data sync polish — A pass of under-the-hood fixes so screens that read from shared data (payments, customer balances, referrals) always show the same number, whether you land on them from the dashboard, POS, or a customer profile.
Cleaner sample data — No customer starts with both class credit and account credit at the same time — matches the studio's new single reward-type rule. On refresh, existing sessions reset to this cleaner starting state.
