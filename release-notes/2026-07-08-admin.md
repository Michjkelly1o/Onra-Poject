Note:
Today's feedback from the meeting/docs is not tackled yet and pushed to the Vercel version today, we only push what we have done this week and the previous week. In the prototype, maybe you will see some 404 pages; that means the page/module/feature is still being developed.
New side for admin: to navigate to admin please change the link to "https://onra-poject.vercel.app/admin/dashboard" and for the instructor side, please change it back to "https://onra-poject.vercel.app/instructor/dashboard"
Interactive experience — This is a fully interactive prototype. Feel free to click, create, edit, and delete items to test the workflows.
Live updates — The prototype uses realistic sample data. Any changes you make apply live as you navigate between different screens.
Session persistence — Any data you create, edit, or cancel survives a page refresh and tab close. Your demo session sticks until you explicitly reset.

1 Dashboard
The daily command center — today's classes, what needs attention, and quick stats for the studio.
Capabilities:
Today / Coming Up / Performance tabs — The dashboard is organized into three tabs, and the tab bar now stays fixed at the top as you scroll down.
Needs attention today — Back in the Today tab, with a tighter, strictly today-scoped list and more of the items that need action surfaced.
Win-back message — A new pop-up to compose and send a win-back message to at-risk clients, ready to send.
Add widget, centered — The Add widget dialog now opens in the middle of the screen.
Under-filled classes tidy-up — Under-filled classes no longer clutter "Needs attention today"; in the Coming Up tab's under-filled list, the "Promote class" action is hidden.

2 Promotions
Discount codes you create and manage for checkout.
Capabilities:
Renamed to "Promotions" — Every "Promo code" label across the app now reads "Promotions" to match your wording.

3 Roles & Permissions
Define what each staff role can and can't do.
Capabilities:
Roles exist once — A role is a single permission set; you no longer see a separate copy per branch. The old branch column is gone.
Branch chosen at assignment — You pick the branch when you assign a person to a role, not when the role is created.
Unique role names — Every new role must have a distinct name, so roles are always tellable apart in the list.

4 Settings
Studio configuration across Business, Operations, and Customer sections.
Capabilities:
Section tabs stay fixed — The top switcher (Business / Operations / Customer) that changes the settings view now stays pinned as you scroll.
Branding as a side panel — Customizing your brand design now slides in from the side instead of taking over a full page.

5 Insights
Performance metrics across finance, clients, classes, and marketing.
Capabilities:
Refreshed data — Updated the underlying sample data so the Insights cards and charts read accurately.

General updates
Quick actions on cards — Class template, promotion, and campaign cards now have a three-dot (⋮) menu for quick actions.
Stability pass — Reviewed the admin side end to end so information stays in sync across screens as you create, edit, and delete.
