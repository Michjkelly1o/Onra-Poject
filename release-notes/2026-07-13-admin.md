Note:
This push covers the client's follow-up feedback from the last review. In the prototype, maybe you will see some 404 pages; that means the page/module/feature is still being developed.
New side for admin: to navigate to admin please change the link to "https://onra-poject.vercel.app/admin/dashboard" and for the instructor side, please change it back to "https://onra-poject.vercel.app/instructor/dashboard"
Interactive experience — This is a fully interactive prototype. Feel free to click, create, edit, and delete items to test the workflows.
Live updates — The prototype uses realistic sample data. Any changes you make apply live as you navigate between different screens.
Session persistence — Any data you create, edit, or cancel survives a page refresh and tab close. Your demo session sticks until you explicitly reset.

1 Dashboard
The owner's daily overview — today's activity, performance, and the customers and classes that need attention.
Capabilities:
Welcome Line Only On First Tab — The "Welcome, Forma Studio" greeting now shows only on the first tab. It's been removed from the Performance tab so the header stays clean when you switch views.
More Breathing Room Under The Title — Added extra spacing between every page title and the breadcrumb underneath, so headers no longer feel crowded.

2 Business & locations (Branches)
Studio locations, each with their own hours, rooms, and now their own timezone.
Capabilities:
Timezone Follows The Branch — When you add or edit a branch, the timezone is now derived automatically from the country / state / city you pick — no manual timezone dropdown to get wrong. Dubai branches read Asia/Dubai, Riyadh branches read Asia/Riyadh, and so on. Every screen that shows a class or appointment time now tags it with the branch's timezone so an owner running multi-city branches never has to guess which zone a 9 AM class is in.

3 Class schedule
The full week / month / list view of every class across every branch.
Capabilities:
New Recurring Ends Options — When you create or edit a recurring class, the "Ends" section now offers three choices on one row: Never / On [date] / After [N] classes. Matches the Elly reference exactly. Picking "On" opens a date picker; picking "After" opens a small number input.

4 Customer notifications
Every automated message that goes out to members — booking confirmations, class reminders, payment receipts, marketing.
Capabilities:
Delivery Hours Modal Redesigned — The quiet-hours setting is now a single row: "Pause messages between [start] and [end]" with the on/off toggle beside it. Simpler to configure and easier to explain to staff.
Admin ↔ Customer Preferences Wired — Any change a customer makes to their notification preferences on the customer app now reflects instantly in the admin's customer detail page (and vice-versa). Same data source on both sides — no more drift between what the customer set and what admin sees.

5 Branding
Studio identity — logo, colours, typography, portal preview.
Capabilities:
Portal Preview Now Uses The Real Customer App — The Login / Home / Class preview tabs inside "Customize design settings" now render the actual customer screens, not a mockup. Change the primary colour, the tertiary tint, the typeface, or upload a logo, and the preview updates live as you scrub. Whatever you see is exactly what the customer will see once you hit Save.

6 Class schedule details / Service details
The customer roster inside a specific class or service booking.
Capabilities:
Context Pills On Each Customer — Each customer row on the class roster and the appointment / service roster now shows small badges next to their name so an admin or instructor can spot who's who at a glance. The badges are system-derived and prioritised — up to 2 show per row:
Birthday — the customer's birthday falls on the class / appointment date.
1st class / 1st appointment — this is their very first booking.
Milestone — 10th, 25th, 50th, 100th, 250th, 500th, or 1000th class / appointment.
New Member — they joined within the last 30 days.
