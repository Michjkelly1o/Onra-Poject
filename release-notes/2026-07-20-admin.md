Note:
This push tightens the Dashboard's Today, Coming Up, and Widgets sections after last week's client feedback, folds the class-type filter into the toolbar on Class Schedule, cleans up the Add customer form, and adds a new Migration & imports page under Settings → Operations. In the prototype you may still see some 404 pages; that means the page/module/feature is still being developed.
New side for admin: to navigate to admin please change the link to "https://onra-poject.vercel.app/admin/dashboard" and for the instructor side, please change it back to "https://onra-poject.vercel.app/instructor/dashboard"
Interactive experience — This is a fully interactive prototype. Feel free to click, create, edit, and delete items to test the workflows.
Live updates — The prototype uses realistic sample data. Any changes you make apply live as you navigate between different screens.
Session persistence — Any data you create, edit, or cancel survives a page refresh and tab close. Your demo session sticks until you explicitly reset.

1 Dashboard
The three-tab dashboard — Today, Coming Up, and Performance.
Capabilities:
Merged Type + Locations filter — The session-type filter and the locations filter on Today and Coming Up now sit inside one shared dropdown, matching the pattern used on the Performance tab. Type is single-select, locations are multi-select, and picking "All locations" toggles every branch together.
Needs Attention renamed — The "Needs attention today" section is now just "Needs Attention" — cleaner heading, same behaviour.
Cards moved between sections — At-risk clients moved back to the Today section. The Occupancy card moved into Coming Up and was renamed Capacity used, so it reads as the coming week's utilisation. Trial-ending alerts moved into the Today section under Needs Attention.
Trial ends opens a details modal — Clicking the "Trials end" row in Needs Attention opens a modal listing every member whose trial ends within seven days, with plan, days remaining, and a direct link to the customer profile so you can act on it right away.
Cleaner widget layout — Every widget lost its subtext line, some titles were renamed for clarity, and duplicated widgets were merged so each metric appears once. The dashboard now reads as a scannable grid instead of a stack of labelled paragraphs.
Six widget categories — The Widgets picker now covers Financial, Customer, Class, Private session, Recovery, and Marketing, with new widgets added under each. The Add widget flow lets you preview and drop any of them into the layout without leaving the dashboard.

2 Class Schedule
Studio-wide class schedule with location + session-type filters.
Capabilities:
Filter sits with the other controls — The filter pill now sits inline with All locations, Search, Export, and the "+ Add new" button, so the toolbar reads left-to-right in the same order used everywhere else in admin.
"+ Add new" button — The old "+ Add class" button is renamed "+ Add new" so it stays accurate as we add private and recovery session creation to the same flow.
Simpler search placeholder — "Search class..." is now just "Search..." — matches the other lists in admin.

3 Customer
Customer list, profile, and add flow.
Capabilities:
Cleaner Add customer form — The "Customer details" and "Address details" section titles have been removed from the Add customer form. The fields still group visually on their own, so the extra headings weren't adding anything and made the form feel longer than it is.

4 Settings
Studio-wide policies + operations.
Capabilities:
New Migration & imports page — A new page under Settings → Operations lists the history of every data import into your studio — customers, memberships, packages — with the file name, uploader, row counts (imported vs skipped), status, and a download link to the invalid-rows report where relevant. Filter by date range and status. The "+ Import" button will open the AI-Agent uploader in an upcoming release.

General updates
"Members" is now "Customers" everywhere — Every screen that said "Members" now says "Customers", so admin uses one word for the same thing across the whole app. No more mixing the two.
Full admin audit pass — A cross-module sweep to catch stale copy, off-by-one counts, and mis-labelled columns from prior weeks. Nothing headline-worthy on its own, but the small paper-cuts across admin should be gone.
Customer app updates merged in — This push also folds in the customer-side work from the customer-experience branch (waitlist claim-to-accept flow, live class status, spot-layout parity with the admin room layout, per-branch push toasts). Admin behaviour is unchanged, but the two prototypes now share one data floor — so a booking made on the customer app shows up on the admin roster in the same spot the customer picked.
