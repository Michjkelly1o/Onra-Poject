Interactive experience — This is a fully interactive prototype. Feel free to click, create, edit, and delete items to test the workflows.
Live updates — The prototype uses realistic sample data. Any changes you make apply live as you navigate between different screens.
Session persistence — Any data you create, edit, or cancel survives a page refresh and tab close. Your demo session sticks until you explicitly reset.

1 Services & Appointments
Service detail pages now handle bookings the same way the schedule view does, and the customer roster on each appointment follows the same status rules as a class.
Capabilities:
Appointment list appears on first booking — Service detail pages stay clean until at least one customer books. The moment a customer books, the appointment shows up in the table — same logic the schedule view uses for classes.
Customer status column matches class schedule — Upcoming appointments hide the status column entirely. Ongoing appointments show the column with blank cells until you mark a customer Present. Completed appointments show Present or No-show on every row, just like a class schedule.

2 Business & Locations
Simplified the working-hours setup when creating or editing a branch.
Capabilities:
No more per-day blocked-time slot — When adding a branch, you only set open and close hours for each day. The previous "blocked time" mid-day slot has been removed from branch setup, from admin schedule, and from the instructor schedule too. Fewer fields, less confusion.

3 Customer & Schedule
Waitlist position is now visible everywhere a customer's upcoming class appears.
Capabilities:
Waitlist number on the badge — Upcoming class cards in the customer module now show "Waitlist #3" instead of just "Waitlisted", so you can see how close a customer is to being promoted. Synced across every related view that surfaces a waitlist entry.

4 Staff & Permissions
Roles are no longer locked to preset permissions. You can now build a custom role with custom permissions from scratch.
Capabilities:
Editable permissions on role creation and edit — Previously, the permissions section was locked when creating or editing a role. Now you can fully customize the permission set, supporting cases like "create an Operator role with custom permissions." All cross-module references to roles continue to work with the new custom permission setup.

General updates
Click any table row to open — Every list in admin (customers, products, memberships, gift cards, agreements, branches, instructors, payroll, and more) now opens the detail page when you click anywhere on the row. The kebab menu still works for the row's other actions.
Back returns where you came from — Every detail page now remembers where you opened it from. Close it and you go back to that exact page, not always the module list. For example: click a class card from the Dashboard's "Today's classes" and close it — you land back on the Dashboard, not the Schedule.
Schedule and Staff & Shift fill the screen — Both the schedule grid and the Staff & shift module now fill the available screen height. Only the inner table scrolls. No more double scrollbars or fixed-height cards that crop on a tall screen.
Global search is now an icon button — Moved global search from the wide search bar into a small magnifying-glass icon next to notifications. Click the icon to open the search panel. Saves header space.
Consistent floating bulk-action bar — When selecting multiple rows for a bulk action, the floating bar now uses the same width and layout on every screen.
Removed "Applicable on" card from membership and package detail — The reverse-link card that listed which class templates and services accept a membership has been removed per Figma. You still pick which memberships or packages are accepted from the class template and service side — only the duplicate display was removed.
