Note:
This push lands the Freeze policy rebuild across admin and customer, rebuilds the Dashboard's Coming Up tab against the new client mockup, adds the recurring flow polish on the schedule form, and does a full sweep to make every list toolbar icon-only with hover tooltips. In the prototype you may still see some 404 pages; that means the page/module/feature is still being developed.
New side for admin: to navigate to admin please change the link to "https://onra-poject.vercel.app/admin/dashboard" and for the instructor side, please change it back to "https://onra-poject.vercel.app/instructor/dashboard"
Interactive experience — This is a fully interactive prototype. Feel free to click, create, edit, and delete items to test the workflows.
Live updates — The prototype uses realistic sample data. Any changes you make apply live as you navigate between different screens.
Session persistence — Any data you create, edit, or cancel survives a page refresh and tab close. Your demo session sticks until you explicitly reset.

1 Dashboard
The three-tab dashboard — Today, Coming Up, and Performance.
Capabilities:
Welcome, Forma Studio — The dashboard title now greets the studio by name at the top of the page. Editing the studio name in Settings → Branding updates the greeting instantly.
Toolbar frozen with the tabs — The tab bar and its per-tab controls (type filter, locations, date range, add widget, export) now sit on one row that stays pinned at the top when you scroll — every control stays reachable while you scan the metrics.
Global search moved to the dashboard only — The magnifying glass in the header now only appears on the dashboard. Every other page has its own scoped search, so the extra button was redundant elsewhere.
Report renamed to Export — The Report button on the Performance tab is now Export, uses the same white chrome, and shows the download icon that every other list uses. Same CSV / PDF / Excel picker inside.
Add widget promoted to the main action — The Add widget button on the Performance tab is now the green primary button and sits on the far right so it reads as the main action.
Coming Up rebuilt — The Coming Up tab now shows a KPI tile strip that adapts to the picked session type, a stacked revenue chart per day (or per week over 30 days) split by session type, and a capacity heatmap showing how full each type is per period. Every block respects the type + locations + range picker at the top. Click a bar or a heatmap cell to jump to the schedule for that day.

2 Class Schedule
Studio-wide class schedule with location + session-type filters.
Capabilities:
Hide the class image in the list view — The row thumbnail was noise at list density (the category tag and name already identify the session); the list is cleaner and denser without it. The card view keeps its cover as before.
Multi-select Type filter — The Type section in the Filter panel now lets you pick any combination of Classes, Private, and Recovery so the schedule can show two or three session types at once.
New session data with Ongoing and Upcoming statuses — Eight fresh classes were added starting tomorrow across morning, afternoon, and evening slots. Every one already has customers booked so you can test check-in and roster flows.
Recurring ends is now a vertical stack — In the Add schedule form, the "Never / On date / After N classes" options now stack one per row instead of squeezing onto one line, so the date picker and count input have room to breathe.
Auto-fill time across selected weekdays — In the Add schedule recurring flow, picking a start time on the first day now auto-fills the same time on every other selected weekday whose slot is still empty. If a picked time is unavailable on a specific weekday (branch closed, instructor unavailable, conflict), that day stays as "Select time" so you know to pick manually — every other day still auto-fills.

3 Private sessions & Recovery & wellness
The two session-type modules that share the same list, scoped by session type.
Capabilities:
Total label reflects the module — The "Total N services" line at the top of the Private sessions list now reads "Total N private sessions" and the Recovery & wellness list reads "Total N recovery & wellness". The generic "services" label stays on the umbrella Services list.
Detail page title reflects the type — Opening a private session detail now shows "Private session details" as the page title (and breadcrumb). Recovery & wellness detail shows "Recovery & wellness details". Both replace the older "Service details" title.
Add new button hidden on scoped views — The green Add new button now hides on Private sessions and Recovery & wellness lists so there's a single, consistent entry point for creating them.

4 Settings
Studio-wide policies + operations.
Capabilities:
Freeze policy rebuilt — The Freeze policy panel under Booking rules now covers the full freeze workflow. Billing during a freeze picks between Pauses (payment date shifts) and Stays on schedule (payment date stays, next charge is reduced by the frozen days). Who can freeze picks between Members & admins, Members request-admins approve, and Admins only. New Minimum freeze duration input, per-reason exceptions (no maximum duration / doesn't count toward the yearly limit / waives the freeze fee) — Medical ships with all three on out of the box.
Freeze approval flow — When Who can freeze is set to "Members request, admins approve", the customer's plan card reads "Request freeze" instead of "Freeze". Requests land as a Freeze requested pill on the customer detail Plan tab; a Review freeze request modal shows the requested dates + reason with an Approve or Decline (with an optional note) pair. Approving fires the freeze; declining sends the note back to the customer.
Freeze notifications — When a customer freezes, both the customer bell and the admin bell fire. Three days before the freeze end date, the customer gets a heads-up ("Freeze ending soon"). At the end date, the plan auto-resumes; both sides get notified again.
Frozen plans block new bookings — While a plan is frozen, class booking, appointment booking, waitlist claim, and admin walk-in booking all show a clear "You can book again on <date>" message and disable the confirm button. Existing bookings on the frozen plan stay valid.
Per-branch overrides for marketing notifications — Under Customer notifications, the marketing rows (Studio announcements, New class launch, Special offers, Promo code offers) now accept per-branch overrides so a marketing message can go out to a subset of locations. The main row's channel toggles stay the default; overrides fine-tune specific branches.
Breadcrumbs include the settings group — Every Settings sub-page now reads with its group tier — for example "Settings › Operations › Migration & imports" instead of just "Settings › Migration & imports". Same for Business & locations, Branding, Tax, Integrations, Customer notifications, and Agreements. Referral program and Account settings sit outside the tabbed groups and skip the group crumb by design.
Referral Overview date filter — The Referral program's Overview tab now has a date period filter in a toolbar above the containers, so you can scope the overview KPIs to a specific window instead of always looking at all-time totals.
Total Sales report renamed — On the Total sales report, the title dropped the "(orders)" suffix and the grouping dimension previously called "Revenue category" is now "Sale category" — matches the actual grouping semantic.

General updates
Icon-only Filter, Search, and Export across every list — Every admin and instructor list toolbar (Customers, Class schedule, Products, Gift cards, Promotions, Marketing, Class templates, Categories, POS, Compensation, Payroll, Roles & permissions, Staff & shift, Business & locations, Tax, Agreements, Integrations, Insights, KPI, Instructor Schedule + Earnings, and every detail page with a roster or bookings table) now shows Filter, Export, and Search as square icon buttons. Hover shows a tooltip so admins know what each does. Search expands to an input when clicked and collapses back to an icon on blur — every other button stays icon-only.
Sticky dashboard chrome — The dashboard tabs and per-tab toolbar sit on the same row and freeze together at the top of the page when you scroll.
Booking guards on frozen memberships — Any new booking attempt (class, appointment, waitlist claim, admin walk-in) on a customer whose membership is currently frozen is blocked with a clear "can book again on <date>" message. Existing bookings and history are untouched.
Consistent tooltip vocabulary — Every hover tooltip in admin (toolbar buttons, chart bars, capacity cells, truncated service names) uses the same dark chrome so the app speaks one visual language.
