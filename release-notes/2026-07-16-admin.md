1 Dashboard
The three-tab dashboard — Today, Coming Up, and Performance.
Capabilities:
Type filter as pills — The session-type filter on Today is a horizontal pills row (All / Class / Private / Recovery) instead of a dropdown. Selected pill uses the same mint highlight as the multi-select filter language across the app.
Coming Up gets a type filter — Same pills as Today, so each session type shows its own card set. All shows Revenue / Bookings / At-risk clients / Expiring credits. Classes adds Recurring revenue, Expiring memberships, Under filled classes, Trials ending, and Occupancy. Private and Recovery show the slots-based Occupancy card.
Renamed cards — "Upcoming recurring revenue" reads as "Recurring revenue", "Bookings ahead" reads as "Bookings", "Renewals due" reads as "Expiring memberships".
Real delta chips — The green / red percentage change on Revenue, Recurring revenue, and Bookings now compares against the prior N-day window (matches the range pill). Was a fixed placeholder before.
Payments collected over time — The Payments-collected widget carries the failed-payments count as a red chip in its header and small red bars under the collected line. Click the chip to open the same Failed payments list the Coming Up tab uses; totals always match.
Payments by source — Renamed from "Payments collected by source" for clarity. The other two payment widgets (status and by method) are consolidated into the merged Payments collected widget, so the finance corner of the picker is simpler.
Intro to member funnel — Redrawn as three horizontal labeled bars (Tried an intro, Returned, Bought a plan) sized by client count. Bars can never exceed the previous stage, and the "% came back" / "% converted" pills between them show the drop-off cleanly.
Add widget placeholder is polished — Sits beside the last widget when there's an empty column, drops to its own row when the grid is even. Widget cards in the same row now match heights end-to-end.

2 Point of Sale
Checkout for memberships, packages, and gift cards.
Capabilities:
Referral credit at checkout — A member's account credit (from referrals, refunds, or manual grants) now appears as a toggle above the payment method picker. Turning it on applies the balance as a reduction — like a discount — and the remainder goes to the chosen payment method. The Wallet is no longer a standalone payment method.
Consistent totals across screens — Cart total on the main POS page and the checkout page now use the same math order: Subtotal → Tax → Discount → Account credit → Total. What the cart shows is what the receipt shows.
Tax on the receipt — The receipt now always shows the tax row, matching the transaction record on the customer's payment history.
Icon matches payment cards — The Account credit tile uses the same 32×32 white-on-gray chrome as the payment method cards, so the two sections read as one visual language.
Refunds restore the credit — Refunding a sale that used account credit returns the credit to the customer's balance. The balance shown on the referral tab, on POS next time, and in the customer app all reflect the change instantly.

3 Customer
Member profiles + list.
Capabilities:
Wallet tab removed — The stand-alone Wallet tab on the customer profile is gone. Account credit is now part of the Referrals tab.
Referrals tab shows live balance — The Account credit cell on the Referrals card shows the customer's live spendable balance, so the same number a cashier sees on the POS toggle appears here.
Import button hidden — The "Import data" button on the customer list is hidden pending a proper migration flow build. Nothing else on the list changed.

4 Class Schedule
Class detail — roster and attendance.
Capabilities:
One-click Present — On an Ongoing class, "Present" is a direct button in the row, not hidden inside the kebab. Click once and the button flips to a mint "Present" badge. A no-show is auto-flagged by the system, so there's no manual no-show button.
Cleaner confirm modal — The bulk "Mark customers as present" confirmation uses the same primary button color as every other primary action in the app (matches "Add class").
Service side matches — The private and recovery appointment detail pages have the same inline Present button + bulk confirmation modal, so class and appointment attendance flows feel identical.

5 Compensation & Payroll
How the studio pays its team.
Capabilities:
Earnings match everywhere — The Earnings column on the compensation list, the payout column on Run payroll, and the Total earnings on Staff details all read the same number for the same staff, period, and branch. Same rule in the CSV exports.
Real "Class revenue base" — The Class revenue base metric card sums the actual revenue from payroll entries instead of an estimated multiplier.
Paid stays paid — Marking a Front Desk or Branch Admin as paid on Run payroll now correctly shows "Paid" on the compensation list too — the two screens no longer disagree.
Refunds affect commission — A live refund of a POS sale now removes that sale from the seller's commission base, so next payroll's commission number reflects the actual net.
Instructor earnings caption — The "vs …" line under the earnings KPI on the instructor side matches the selected period ("vs last month" when the picker says This month, etc.) instead of always saying "vs last week".

General updates
Dashboard heights aligned — Cards in the same row on the Performance tab stretch to match heights, so a shorter widget no longer leaves a visible white gap next to a taller one.
Coming Up counts include today — Cards now correctly include rows dated today (previously dropped by a timezone quirk), and clicking through opens a list with the same count.
Expiring memberships modal matches the card — The list opened from the Expiring memberships card shows exactly the members the card counted, no more no less.
